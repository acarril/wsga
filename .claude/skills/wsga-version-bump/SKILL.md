# wsga Version Bump

All wsga components (R package, Stata ADO files, Stata help files, package metadata)
share a single unified `MAJOR.MINOR.PATCH` version. This skill updates every location
atomically so nothing drifts.

## Step 1 — Read the current version

```bash
grep "^Version:" DESCRIPTION
```

This is the authoritative source. Use whatever `X.Y.Z` it returns as `OLD_VERSION`.

## Step 2 — Determine the new version

Ask the user which component to bump (or infer from context):

| Type | When | Example |
|------|------|---------|
| **patch** (default) | Bug fixes, docs, Stata-only changes | 1.0.1 → 1.0.2 |
| **minor** | New features, new options, API additions | 1.0.2 → 1.1.0 |
| **major** | Breaking changes, removed functionality | 1.1.0 → 2.0.0 |

Compute `NEW_VERSION` by incrementing the appropriate component and zeroing all
lower components (e.g. minor bump: `1.0.2 → 1.1.0`, not `1.0.2 → 1.1.2`).

Get today's date:

```bash
TODAY=$(date +%Y-%m-%d)      # for version banners: 2026-05-11
TODAY_COMPACT=$(date +%Y%m%d) # for Distribution-Date fields: 20260511
```

## Step 3 — Update every version string

Run all substitutions from within `rddsga-repo/`. Replace `OLD_VERSION` and dates
where indicated. Use `sed -i ''` on macOS.

### R package

```bash
# DESCRIPTION — Version field
sed -i '' "s/^Version: .*/Version: NEW_VERSION/" DESCRIPTION

# NEWS.md — prepend new section (see Step 4 for content)
```

### Stata ADO files — `*!` banner line

```bash
sed -i '' "1s|^.*|*! NEW_VERSION Alvaro Carril TODAY|" stata/wsga.ado
sed -i '' "1s|^.*|*! NEW_VERSION Alvaro Carril TODAY|" stata/rddsga.ado
```

### Stata help files — `{* *! version ...}` line

```bash
sed -i '' "2s|{\\* \\*! version .*}|{* *! version NEW_VERSION TODAY}{...}|" stata/wsga.sthlp
sed -i '' "2s|{\\* \\*! version .*}|{* *! version NEW_VERSION TODAY}{...}|" stata/wsga_rdd.sthlp
sed -i '' "2s|{\\* \\*! version .*}|{* *! version NEW_VERSION TODAY}{...}|" stata/wsga_did.sthlp
sed -i '' "2s|{\\* \\*! version .*}|{* *! version NEW_VERSION TODAY}{...}|" stata/rddsga.sthlp
```

### Stata package metadata files

```bash
# wsga.pkg — v line (line 1) and Distribution-Date
sed -i '' "1s|^v .*|v NEW_VERSION|" stata/wsga.pkg
sed -i '' "s|d Distribution-Date: .*|d Distribution-Date: TODAY_COMPACT|" stata/wsga.pkg

# rddsga.pkg — same
sed -i '' "1s|^v .*|v NEW_VERSION|" stata/rddsga.pkg
sed -i '' "s|d Distribution-Date: .*|d Distribution-Date: TODAY_COMPACT|" stata/rddsga.pkg

# stata.toc — v line (line 1)
sed -i '' "1s|^v .*|v NEW_VERSION|" stata/stata.toc
```

> **Never touch**: `version 11.1` lines inside `.ado` files (Stata minimum
> compatibility version) or `RoxygenNote:` in `DESCRIPTION` (auto-generated by
> roxygen2). These are not the package version.

## Step 4 — Write the NEWS.md entry

Collect the commits since main branched off:

```bash
git log main..HEAD --oneline
```

Prepend the following block at the **very top** of `NEWS.md`, above all previous
releases. Fill in the bullet points from the commit log — one logical change per
line, in plain English:

```markdown
## wsga NEW_VERSION (TODAY)

### [Bug fixes / New features / Breaking changes — use whichever apply]

- <bullet from commit log>
- <bullet from commit log>

---

```

If the user already drafted NEWS.md bullets in a prior commit message or the PR
description, use those rather than rewriting.

## Step 5 — Verify nothing was missed

```bash
grep -rn "OLD_VERSION" \
  --include="*.R" --include="*.ado" --include="*.sthlp" \
  --include="*.pkg" --include="*.toc" --include="*.md" \
  --include="DESCRIPTION" .
```

Expected result: only hits inside old `## wsga OLD_VERSION (...)` headers in
`NEWS.md` (historical entries). Any hit outside `NEWS.md` means a location was
missed — fix it before proceeding.

## Step 6 — Stage all version files and report

```bash
git add DESCRIPTION NEWS.md \
  stata/wsga.ado stata/rddsga.ado \
  stata/wsga.sthlp stata/wsga_rdd.sthlp stata/wsga_did.sthlp stata/rddsga.sthlp \
  stata/wsga.pkg stata/rddsga.pkg stata/stata.toc
```

Tell the user:
- **OLD_VERSION → NEW_VERSION** (date: TODAY)
- List of files updated
- The NEWS.md snippet that was prepended

Then continue with `gh pr create` (or return to the calling workflow if invoked
from `superpowers:finishing-a-development-branch`).
